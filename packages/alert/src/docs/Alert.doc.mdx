# Alerts
Display an alert with text to users.

[Github repo](https://github.com/unicoderns/ui/tree/master/packages/alert)

```html
    <ui-alert :="args" :show="toggle" @show="show" @close="close" @hide="hide">{{ args.default }}</ui-alert>
```

## Props
| Name | Type  | Description |
| ---- | ---- | ----------- |
| `animate` | `boolean` | add animate class to alert |
| `dismissible` | `boolean` | show close button |
| `show` | `boolean` | show/hide the alert |
| `variant` | `string` | variant class defined in the theme |
| `invert` | `boolean` | inverts the close button |


## Slots
| Name | Description |
| ---- | ----------- |
| `default` | actual alert content |

## Events
Alert will emit this events while changing state

| Event | Params | Desription |
| ---- | ---- | ----------- |
| show | NONE | Will only raise after transition |
| close | NONE | Will only when closing |
| hide | NONE | Will only after closing animation |


## Accessibility
Use the following properties to change the default accessibility text

| Name | Property | Default |
| ---- | ---- | ----------- |
| role | aria:role | `alert` |
| buttonClose | aria:buttonClose | `close` |

## Theme
Use the next `json` to define a new theme for the component

```json
{
  cssClass: {
    main: string // main alert class
    dismissible: string // class for dismissible state
    animated: string // class for animated alerts
    variants: KeyPairString // define a custom list of variants
    components: {
      buttonClose: string // class for button close
    }
  }
}
```

Check the page [what is a theme](/docs/docs-theming--page#what-is-a-theme) to see how to use this json


## Inner components

Note that the Alert uses [UiTransition](?path=/docs/ui-transition-default&args=#transitions) and [UiButtonclose](?path=/docs/ui-closebutton--default&args=#close-button) components internally, so when providing a theme you should provide for them as well.
